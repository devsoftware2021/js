JavaScript Functions

1. Function Declaration

A named function that can be called before its definition.

function greet(name) { return "Hello, " + name + "!"; } console.log(greet("Alice")); // Output: Hello, Alice!

2. Function Expression

A function assigned to a variable. These are not hoisted.

const greet = function(name) { return "Hello, " + name + "!"; }; console.log(greet("Bob")); // Output: Hello, Bob!
// Error: Cannot call a function expression before it's defined console.log(greet("Charlie")); // ReferenceError: greet is not defined const greet = function(name) { return "Hello, " + name + "!"; };

3. Arrow Functions

A concise way to write functions. Does not bind its own `this` context.

const greet = (name) => { return "Hello, " + name + "!"; }; console.log(greet("Dave")); // Output: Hello, Dave!

For single-expression functions, the `return` keyword can be omitted:

const greet = name => "Hello, " + name + "!"; console.log(greet("Eve")); // Output: Hello, Eve!

4. Callback Functions

Passing a function as an argument to another function.

function processUserInput(callback) { const name = "Frank"; callback(name); } processUserInput(function(name) { console.log("Hello, " + name + "!"); // Output: Hello, Frank! });

Example Combining Concepts

A real-world use case combining function types:

// Arrow function as a callback in an array method const numbers = [1, 2, 3, 4, 5]; const squared = numbers.map(num => num * num); console.log(squared); // Output: [1, 4, 9, 16, 25]
// Error: Missing parameters in an arrow function const greet = name => "Hello!"; // Incorrect call console.log(greet()); // Output: Hello undefined!
