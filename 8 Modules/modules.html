JavaScript Modules

1. ES6 Modules

ES6 Modules allow you to divide your JavaScript code into reusable pieces.

  • Import: Bring functionality from another file.
  • Export: Share functionality from the current file.

Example: Exporting Functions and Variables

/* File: mathUtils.js */
export const add = (a, b) => a + b;
export const subtract = (a, b) => a - b;

// Default export
export default function multiply(a, b) {
    return a * b;
}

Example: Importing in Another File

/* File: app.js */
import multiply, { add, subtract } from './mathUtils.js';

console.log(add(2, 3));         // Output: 5
console.log(subtract(5, 3));    // Output: 2
console.log(multiply(4, 3));    // Output: 12

Important Notes:

  • Use the type="module" attribute in HTML when including ES6 module scripts:
  • ES6 modules are supported in modern browsers and Node.js (version 12+).

2. CommonJS Modules

CommonJS Modules are used in Node.js for managing dependencies and modularizing code.

  • require(): Import modules in Node.js.
  • module.exports: Export functionality in Node.js.

Example: Exporting Functions and Variables

// File: mathUtils.js
const add = (a, b) => a + b;
const subtract = (a, b) => a - b;

const multiply = (a, b) => a * b;

// Export functions
module.exports = {
    add,
    subtract,
    multiply,
};

Example: Importing in Another File

// File: app.js
const { add, subtract, multiply } = require('./mathUtils');

console.log(add(2, 3));         // Output: 5
console.log(subtract(5, 3));    // Output: 2
console.log(multiply(4, 3));    // Output: 12

Key Points:

  • CommonJS is the default module system in Node.js.
  • Use require() to import and module.exports to export functionality.
  • CommonJS does not work natively in browsers but can be bundled using tools like Webpack.

Summary: ES6 vs. CommonJS Modules

Feature ES6 Modules CommonJS Modules

Import Syntax import require()
