Basic Debugging in JavaScript

Debugging is essential for finding and fixing errors in your code. Below are three common methods for debugging JavaScript.

1. Using console.log() for Debugging

The console.log() method prints messages or variable values to the browser’s or Node.js’s console, making it the simplest way to understand what's happening in your code.

Example

function add(a, b) { console.log("Inputs:", a, b); // Log inputs const result = a + b; console.log("Result:", result); // Log result return result; } add(5, 3); /* Console Output: Inputs: 5 3 Result: 8 */

Tips:

  • Use console.warn() for warnings and console.error() for errors.
  • Combine with JSON.stringify() for inspecting complex objects:
console.log("User Data:", JSON.stringify(userData, null, 2));

2. Inspecting Errors: Understanding Stack Traces

When an error occurs, JavaScript generates a stack trace to show the sequence of function calls leading to the error. These are visible in the console.

Example: Common Error

function divide(a, b) { if (b === 0) { throw new Error("Division by zero is not allowed"); } return a / b; } try { divide(10, 0); // This will throw an error } catch (error) { console.error("Error caught:", error.message); console.error("Stack Trace:", error.stack); } /* Console Output: Error caught: Division by zero is not allowed Stack Trace: Error: Division by zero is not allowed at divide (script:2:15) at script:6:5 */

Reading a Stack Trace:

  • Error Message: The description of the error.
  • Function Call Chain: The sequence of calls that led to the error, starting from the most recent.

3. Using the debugger

The debugger statement and browser/Node.js developer tools allow you to pause code execution, inspect variables, and step through code.

In Browsers

  • Open Developer Tools (F12 or right-click > "Inspect").
  • Go to the Sources tab.
  • Set Breakpoints by clicking the line number in your script.
  • Use the Step Over, Step Into, and Step Out buttons to navigate the code.

Example

function multiply(a, b) { debugger; // Execution will pause here return a * b; } multiply(4, 5);

When running this code in the browser console, execution pauses at debugger. You can inspect a and b in the developer tools.

In Node.js

Run your script with the --inspect flag:

node --inspect yourScript.js

Open Chrome and navigate to chrome://inspect.

Click "Open dedicated DevTools for Node" to debug.

Summary of Tools:

  • console.log(): Quick and simple for printing variable values and messages.
  • Error Stack Traces: Help identify where and why errors occur.
  • Debugger: Powerful tool for real-time inspection and stepping through code.
Note: Debugging is an essential skill for writing and maintaining clean, efficient code.
