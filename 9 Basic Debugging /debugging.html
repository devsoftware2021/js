Basic Debugging in JavaScript

Debugging is the process of finding and fixing errors in your code. Below are the most common techniques to debug JavaScript.

1. Using console.log()

The console.log() method prints messages or variable values to the console. It's the simplest way to understand what's happening in your code.

Example: Debugging with console.log()

function add(a, b) { console.log("Inputs:", a, b); // Logs the inputs const result = a + b; console.log("Result:", result); // Logs the result return result; } add(5, 3); /* Console Output: Inputs: 5 3 Result: 8 */

2. Inspecting Errors: Understanding Stack Traces

When an error occurs, JavaScript provides a stack trace that shows the sequence of function calls leading to the error.

Example: Catching Errors

function divide(a, b) { if (b === 0) { throw new Error("Division by zero is not allowed"); } return a / b; } try { divide(10, 0); // This will throw an error } catch (error) { console.error("Error caught:", error.message); console.error("Stack Trace:", error.stack); } /* Console Output: Error caught: Division by zero is not allowed Stack Trace: Error: Division by zero is not allowed at divide (script:2:15) at script 6 5 */

3. Using the debugger Statement

The debugger keyword pauses the code execution, allowing you to inspect variables and step through your code.

Example: Using debugger

function calculate(a, b) { debugger; // Execution pauses here when debugging return a * b; } calculate(4, 5);

When the debugger statement is executed, the browser developer tools or Node.js debugger will pause the code execution.

4. Summary

  • Use console.log() to print variables and understand code flow.
  • Inspect error stack traces to pinpoint where an error occurred.
  • Use the debugger statement for an interactive debugging experience.
Note: Debugging is an essential skill for writing and maintaining clean, efficient code.
